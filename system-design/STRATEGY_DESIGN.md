# 투자 전략 설계

> 일반적인 트레이딩 전략 원칙과 조합을 정의한다.
> 특정 자산에 특화된 전략은 별도 파일에서 다룬다. (구현 완료 → `trader-common/solution-design/archive/XRP_STRATEGY_DESIGN.md`)

---

## 1. 기본 전략 유형

### 1️⃣ 추세 추종 전략 (Trend Following)

> "추세는 생각보다 오래 간다"

- 상승 추세 → 매수 유지
- 하락 추세 → 매도 유지
- 방향에 순응하는 전략

**잘 먹히는 장**: 강한 방향성이 있는 시장. 비트코인 대세 상승/하락 구간

**단점**
- 횡보장에서 계속 손절
- 진입이 늦어질 수 있음

**요약**: 큰 돈은 추세에서 나온다. 하지만 인내심이 필요하다.

---

### 2️⃣ 역추세 전략 (Mean Reversion)

> "가격은 평균으로 돌아온다"

- 과매도 → 매수
- 과매수 → 매도

**잘 먹히는 장**: 횡보장 / 박스권

**단점**: 강한 추세에서 계좌 파괴 위험

**요약**: 단기 승률은 좋지만 손절 관리가 생명

---

### 3️⃣ 돌파 전략 (Breakout)

> "에너지가 쌓인 곳은 크게 터진다"

- 저항 돌파 → 매수
- 지지 이탈 → 매도

**잘 먹히는 장**
- 변동성이 막 커지기 시작할 때
- 뉴스 전후

**단점**: 가짜 돌파(Fake Breakout) 매우 많음

**요약**: 손절은 짧게, 익절은 길게

---

### 4️⃣ 모멘텀 전략 (Momentum)

> "강한 것은 더 강해진다"

- 이미 오르고 있는 자산을 추격
- 하락 모멘텀도 동일

**잘 먹히는 장**: 테마주, 알트 시즌, 강세장

**단점**: 꼭대기에서 잡기 쉬움

**요약**: 빠른 판단 + 빠른 손절 필수

---

### 5️⃣ 패턴 기반 전략 (Price Action)

> "가격 그 자체가 모든 정보를 담고 있다"

- 지표 최소화
- 캔들 + 구조 중심

**잘 먹히는 장**: 모든 시장 (숙련도 의존)

**단점**
- 주관 개입 위험
- 경험이 없으면 재현성 낮음

**요약**: 고수들이 최종적으로 도달하는 영역

---

### 6️⃣ 변동성 전략 (Volatility)

> "방향은 몰라도 크게 움직일 건 안다"

**잘 먹히는 장**: 이벤트 전, 장 시작/마감

**요약**: 방향 예측보다 "움직임"에 베팅

---

### 7️⃣ 이벤트 / 뉴스 기반 전략

> "가격은 뉴스에 반응한다"

주요 이벤트 예시: 금리, ETF 승인, 실적, 규제 뉴스

**특징**
- 변동성 폭발
- 예측보다 대응이 중요

**단점**
- 슬리피지
- 급변 리스크

**요약**: 개인 투자자에겐 가장 어려운 영역

---

## 2. 안정적인 전략 조합 (실전형)

### 핵심 철학

> 추세에서는 추세를 타고,
> 횡보에서는 되돌림만 먹고,
> 변동성 폭발 구간은 피하거나 소액만 건다

### 메인 전략: 중기 추세 추종 (핵심 엔진)

**왜 메인인가?**
코인 시장은 큰 추세가 나올 때 수익의 대부분이 발생. BTC, ETH는 특히 이 성향이 강함.

**구성**
- 타임프레임: 4H ~ 1D
- 지표: 20EMA / 60EMA, 구조 고점·저점
- 규칙 예시
  - 20EMA > 60EMA → 상승 추세
  - 조정 시 분할 매수
  - 추세 붕괴 시 전량 정리

**장점**
- 슬리피지 영향 적음
- 거래 빈도 낮음
- 심리적 안정성 ↑

👉 전체 자금의 50~70%

---

### 보조 전략: 약한 역추세 (되돌림 매매)

> ⚠️ "강한 역추세" ❌ → 추세 안에서의 역추세만 허용

**역할**
- 추세 추종 진입가를 좋게 만드는 역할
- 수익 보조 + 평균단가 개선

**구성**
- 조건: 상위 추세가 명확할 것
- 지표: RSI (30~40 / 60~70), 지지·저항
- 규칙
  - 상승 추세에서 과매도만 매수
  - 하락 추세에서 과매수만 매도

**장점**
- 승률 높음
- 손절 짧음

👉 자금의 15~25%

---

### 필터 전략: 변동성 회피 + 선택적 돌파

> 이 전략의 목적은 "수익"이 아니라 계좌 보호

**회피 조건**
- 거래량 급증
- 뉴스/지표 직후
- 볼린저 밴드 급확대
- → 아무것도 안 함 (가장 고급 전략)

**선택적 돌파** (아주 제한적으로)
- 장기 박스권
- 거래량 동반
- 손절 매우 짧게

👉 자금의 5~10% 이하

---

### 공통 규칙 (절대 포함)

**① 포지션 사이즈 고정**
- 1트레이드 최대 손실: 총 자금의 0.5~1%
- 연속 손실에도 계좌 생존

**② 전략 혼용 금지**
- 한 포지션에 추세 추종 + 역추세 혼용 ❌
- 진입 이유는 반드시 하나

**③ BTC 기준 필터**
- BTC가 박스/급락 중이면 알트 매매 중단
- "알트는 독립적으로 움직이지 않는다"

---

### 전략 비중 요약표

| 역할 | 전략 | 비중 |
|------|------|------|
| 메인 | 중기 추세 추종 | 50~70% |
| 보조 | 추세 내 되돌림 | 15~25% |
| 옵션 | 제한적 돌파 | 5~10% |
| 보호 | 관망 / 현금 | 나머지 |

---

### 가장 많이 망하는 조합 ❌

- ❌ 역추세 + 고레버리지
- ❌ 단타 + 시장가 + 잦은 거래
- ❌ 뉴스 직후 추격
- ❌ 전략 5개 이상 동시 사용

---

## 3. 박스권 역추세 전략 — 상세 설계

### 개요

횡보장에서 가장 높은 승률을 보이는 핵심 전략. 특히 꼬리 흔들기(페이크)가 잦고 횡보가 긴 코인에 유리하다.

**핵심 공식**
```
[박스 확인]
      ↓
[하단 + RSI 낮음] → 매수
[상단 + RSI 높음] → 매도
      ↓
[박스 이탈] → 생각 말고 손절
```

> 이 전략은 "예측"이 아니라 "확률이 높은 구간만 고르는 전략"이다

---

### 박스권을 실제로 그리는 방법 (실전)

**STEP 1: 타임프레임 고정**

- 1H 또는 4H 사용
- ❌ 5분봉, 15분봉 — 잡음 너무 많음

**STEP 2: "여러 번 막힌 가격" 찾기**

- 박스 상단: 가격이 여러 번 올라갔다가 같은 가격대에서 계속 막힘 → 수평선
- 박스 하단: 가격이 여러 번 내려갔다가 같은 가격대에서 계속 반등 → 수평선
- 최소 3번 이상 확인되면 신뢰도 ↑

```
──────────────  ← 박스 상단 (3번 이상 막힘)
│
│
│
──────────────  ← 박스 하단 (3번 이상 반등)
```

**STEP 3: 박스 안에서만 매매**

- 하단 근처 → 매수
- 상단 근처 → 매도
- 중간 → 관망

**STEP 4: 박스 이탈 기준 정하기**

- 1H 또는 4H 캔들의 **종가**가 박스 위/아래에서 마감
- 이탈 확정 → 손절

---

### 이 전략이 망하는 대표적인 실수 3가지

**❌ 실수 ①: 박스가 아닌데 "박스라고 믿고" 들어감**

저점이 계속 낮아지는 것은 박스가 아니라 하락 추세이다.
박스는 위도 막히고 아래도 막히는 게 반복되어야 한다.

**❌ 실수 ②: RSI만 보고 매매함**

RSI 30인데도 계속 하락하는 경우가 많다.

이 전략의 공식은 항상 이것:
```
[박스 하단] + [RSI 낮음]   ← 둘 다 있어야 함
```
RSI 단독 사용 = 실패 확률 매우 높음

**❌ 실수 ③: 박스 이탈인데 "기다려봄"**

"여기서 설마 더 가겠어", "조금만 더 기다려보자" → 계좌를 망가뜨리는 1번 원인

박스를 깨면 이건 되돌림이 아니라 추세 시작일 가능성이 높다. → 즉시 손절

---

### 박스 관련 수치 기준

**몇 번이면 충분한가?**

| 반복 횟수 (4H 기준) | 판단 |
|---------------------|------|
| 1번 | ❌ 무시 |
| 2번 | ⚠️ 관찰 |
| 3번 이상 | ✅ 박스 기준 사용 |

**"근처"란 얼마인가?**

박스 상·하단은 정확한 한 가격(point)이 아니라 **허용 오차 범위(zone)**이다.

- 실전 기준: ±0.3% ~ ±0.7% (메이저 알트 기준)
- 시각적으로: 여러 4H 캔들의 고점/저점이 겹쳐서 몰린 구간 전체
- 박스는 "선"이 아니라 "영역"이다

**꼬리 vs 몸통, 어디까지 인정하나?**

| 반응 유형 | 인정 여부 | 설명 |
|-----------|-----------|------|
| 몸통 2~3번 | ✅✅✅ | 박스 기준으로 충분 |
| 몸통 1번 + 꼬리 1~2번 | ✅✅ | 실전 사용 가능 |
| 꼬리만 2~3번 | ⚠️ | 후보 |
| 꼬리 1번 | ❌ | 의미 없음 |

> **핵심**: 박스를 만들 때는 '꼬리도 참고'하지만, 박스를 버릴 때(손절 판단)는 '몸통(종가)만 본다'

---

### 1H / 4H 캔들과 손절 기준

**캔들이란?**

캔들 1개 = 그 시간 동안의 가격 요약본

```
     │   ← 위 꼬리 (윗꼬리)
     │
┌───┼───┐
│   ■   │  ← 몸통 (시작가 ↔ 끝가)
└───┼───┘
     │
     │   ← 아래 꼬리 (아랫꼬리)
```

- 몸통: 시작가 ↔ 끝가
- 꼬리: 잠깐 찍고 바로 되돌아온 가격 흔적. "그 가격을 시장이 거부했다"는 흔적

**종가란?**

- 종가 = 그 캔들이 끝났을 때의 가격
- 코인은 장이 안 닫히지만, 시간 단위로 끊어서 종가를 만든다
  - 1H 캔들 → 1시간 끝날 때 가격
  - 4H 캔들 → 4시간 끝날 때 가격

**꼬리 vs 종가 이탈 구분**

```
손절 안 해도 되는 경우 (꼬리만):
  박스 하단
  ──────────────
  │
  │   ↓  ← 꼬리만 (종가는 박스 안)
  │
  ──────────────
  → 전략 유지

손절해야 하는 경우 (진짜 이탈):
  박스 하단
  ──────────────
  │
  ──────────────
  ↓  ← 종가가 박스 아래에서 끝남
  → 손절
```

- 꼬리 = 잠깐 찍고 실패한 움직임
- 종가 = 그 시간 싸움의 최종 결과
- 꼬리만 이탈 → 아직 버틴다
- 종가 이탈 → 바로 손절

---

### 1H와 4H 손절 기준 비교

| 기준 | 1H 손절 | 4H 손절 |
|------|---------|---------|
| 반응 속도 | 빠름 | 느림 |
| 손절 횟수 | 많음 | 적음 |
| 평균 손실 | 작음 | 큼 |
| 가짜 이탈 대응 | 약함 | 강함 |

**추천 조합 (꼬리가 많은 코인 기준)**

- 진입: 1H 흐름을 보고 타이밍 포착
- 손절 판단: 4H 구조가 깨졌을 때 판단

> "조금 흔들리는 건 참되, 구조가 깨졌다는 게 확인되면 나간다"

**실전 예시**

| 상황 | 판단 |
|------|------|
| 1H 종가 이탈, 4H 종가 박스 안 | ❌ 아직 손절 안 함 |
| 1H 종가 이탈, 4H 종가도 이탈 | ✅ 즉시 손절 |

---

### 박스 재설정 기준

**박스는 언제 다시 그리나?**

박스는 '시간이 지났다고' 다시 그리는 게 아니라, '구조가 바뀌었을 때만' 다시 그린다.

**재설정 신호 (4H 기준)**

1. **4H 종가 기준 박스 이탈** → 가장 확실한 신호. 이전 박스 무효, 새 박스 탐색 시작.

2. **지지/저항이 역할을 더 이상 못 할 때** → 하단에서 반등이 약해지거나 없어지면 기존 박스 신뢰도 하락.

3. **수렴 삼각형 형성** → 고점은 낮아지고 저점은 높아지면 에너지 압축 구간. 박스 전략 ❌, 돌파 전략 영역으로 전환.

4. **1H는 흔들리는데 4H 구조가 바뀌었을 때** → 진짜 기준은 4H. 4H 기준으로 박스 재정의.

**재설정하면 안 되는 경우 ❌**

- "시간이 좀 지났으니까"
- "최근 10분봉이 많이 움직였으니까"
- "조금만 위로 그리면 아직 박스 같아서"
- → 이건 자기합리화

**박스 재설정 후 1H 박스**

- 박스 재설정의 최종 결정권은 4H에 있다
- 4H 박스를 다시 그릴 때 → 1H 박스도 같이 재정의
- 1H 박스는 독립적으로 재설정하지 않는다

```
[4H 박스 이탈 확인]
        ↓
[4H 박스 재설정]
        ↓
[1H 박스도 새로 정의]
```

---

### 4H / 1H 박스의 역할 구분

| | 4H 박스 | 1H 박스 |
|--|---------|---------|
| 역할 | 전략 유효/무효 판단, 손절 여부, 구조 기준 | 진입 타이밍, 상·하단 미세 조정 |
| 성격 | 헌법 같은 존재 (잘 안 바뀜) | 실행 규칙 (4H 안에서만 의미) |
| 재설정 기준 | 4H 종가 이탈 시 | 4H 박스가 바뀔 때만 같이 바뀜 |

> **1H 박스는 '독립적인 박스'가 아니다. 항상 4H 박스의 하위 구조다.**

---

### 캔들 시간 경계와 완성된 캔들만 사용

**시간은 내가 자르는 게 아니다**

4H 캔들 경계는 거래소가 이미 정해둔 고정 경계이다:
```
00:00 ── 04:00 ── 08:00 ── 12:00 ── 16:00 ── 20:00 ── 24:00
   [4H]      [4H]      [4H]      [4H]      [4H]      [4H]
```

- ❌ "지금부터 4시간 범위" ← 사용 안 함
- ✅ 완전히 마감된 4H 캔들만 사용

**진행 중인 캔들은 사용하지 않는다**

```
예시 (현재 시간 18:20):
  08:00 ~ 12:00  ✅ 완료된 4H
```

### 1H 박스가 없는 경우 (정상 상태)

1H 캔들에서 의미 있는 반복이 없으면 유효한 1H 박스가 없다. 이건 이상한 게 아니라 정상 상태다.

```
✅ 1H 박스 있는 경우:
  4H 박스
  ┌────────────────────┐
  │   1H 박스          │
  │  ────────────      │
  │                    │
  └────────────────────┘

✅ 1H 박스 없는 경우 (정상):
  4H 박스
  ┌────────────────────┐
  │  1H가 흘러만 감     │
  │  ↑   ↑   ↑   ↑     │
  └────────────────────┘
```

1H 박스가 없을 때는:
- ❌ 억지로 1H 박스 만들지 않는다
- ✅ 4H 하단/상단 근처에서만 신중히 접근
- ✅ 진입 빈도 낮춘다
